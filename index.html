What It Does

Sinister Omnipresence is a native macOS app that lets Sinister Printworks publish 3D printing content to every major social media platform simultaneously from a single interface. One post, one click, six platforms.

The app handles everything end-to-end:

  • Compose once — write a caption, attach photos or videos, add hashtags
  • Per-platform overrides — different caption per platform without re-typing
  • Auto-transcoding — each platform gets the correct aspect ratio, codec, and bitrate automatically
  • Schedule — set a future date/time or publish immediately
  • Compatibility checking — live warnings if your media won't work on a given platform before you publish
  • Rate limiting — built-in throttles that match each platform's daily/per-minute API caps
  • Retry logic — automatic retries with exponential backoff on 429/5xx errors; smart failure classification for permanent errors
  • Token management — OAuth tokens auto-refresh before expiry; health-check on startup

Architecture

Native SwiftUI macOS app. No server backend. All OAuth flows run on-device via ASWebAuthenticationSession with PKCE. All API clients are Swift actors for thread safety. Persistence is file-based JSON (no CoreData/SwiftData).

Media Pipeline (Instagram / Facebook)

Instagram requires publicly accessible HTTPS URLs — local file paths don't work. The app uses a Cloudflare R2 staging bucket:

Local media file
  → POST https://sinisterprintworks.net/api/media/social (header: X-Social-Upload-Key)
  → Cloudflare R2 bucket (public CDN URL generated)
  → Meta Graph API uses public URL to create container / upload photo
  → After successful publish, R2 object is deleted (cleanup)

Instagram images are auto-resized to fit within the 4:5 to 1.91:1 aspect ratio window before upload. Facebook always receives the original unmodified image.

Known Issues & Tech Debt

Medium PlatformPublisher.swift is legacy

Raw URLSession.shared + JSONSerialization. PostSchedulerService now uses typed actor API clients directly. PlatformPublisher should be deprecated and removed.

Medium Fake post IDs mask failures

PlatformPublisher returns fb_<UUID> placeholders when API calls fail instead of throwing. This hides errors.

PR Open EngagementService / SocialDispatcher on API v21.0

Data endpoints are v24.0 but these two services still reference v21.0 Graph API URLs. Fix written in PR #7 — not yet merged to main.

PR Open Token expiry "not found" warning

Logs show "Token exists but no expiry found" for Facebook. Silent back-fill fix written in PR #8 — not yet merged to main.

Planned Connection verification (Roadmap Step 3)

No lightweight token health check against live APIs. App reports token state from stored data only, not from a live API ping.

Planned Analytics view is a placeholder

AnalyticsView exists in the UI but has no live engagement data fetching implemented.
